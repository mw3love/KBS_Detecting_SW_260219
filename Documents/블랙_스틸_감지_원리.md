# KBS 방송 모니터링 시스템 — 블랙/스틸 감지 원리

---

## 1. 왜 필요한가?

방송 송출 중 화면이 **까맣게 변하거나(블랙)**, **멈추거나(스틸)** 하면 방송 사고입니다.
사람이 16채널을 동시에 24시간 주시하는 것은 불가능하므로, 프로그램이 자동으로 감지하여 알림을 울립니다.

---

## 2. 블랙 감지 — "화면이 까만가?"

### 기본 원리

화면의 모든 픽셀(점)에는 밝기 값이 있습니다 (0 = 완전 검정, 255 = 완전 흰색).

```
[감지 순서]

1단계. 각 픽셀이 어두운지 판단
       ┌─────────────────────────────────────┐
       │  밝기 임계값 = 5                     │
       │  → 밝기가 5 미만인 픽셀 = "어두운 픽셀" │
       └─────────────────────────────────────┘

2단계. 어두운 픽셀의 비율 계산
       ┌─────────────────────────────────────┐
       │  전체 픽셀 중 어두운 픽셀이 98% 이상?  │
       │  → YES: 블랙으로 판단                 │
       │  → NO:  정상                         │
       └─────────────────────────────────────┘

3단계. 모션 억제 (오감지 방지)
       ┌─────────────────────────────────────┐
       │  블랙으로 판정됐지만 화면에 움직임이    │
       │  있는가? (예: 어두운 장면에 자막 이동)  │
       │  → 움직임 있으면 블랙 판정 취소        │
       └─────────────────────────────────────┘

4단계. 일정 시간 이상 지속되면 알림
       ┌─────────────────────────────────────┐
       │  블랙이 20초 이상 계속되면 알림 발생    │
       └─────────────────────────────────────┘
```

### 시각적 예시

```
  정상 화면                    블랙 화면
 ┌──────────────┐            ┌──────────────┐
 │ ■□□□■□□□■□□ │            │ ■■■■■■■■■■■ │
 │ □□■□□□□■□□□ │            │ ■■■■■■■■■■■ │
 │ □□□□□■□□□□■ │ ← 밝은 픽셀  │ ■■■■■■■■■■■ │ ← 거의 모든 픽셀이 어두움
 │ ■□□□□□□□■□□ │    많음      │ ■■■■■■■■■■■ │    (98% 이상)
 │ □□■□□■□□□□□ │            │ ■■■■■■■■■■■ │
 └──────────────┘            └──────────────┘
  어두운 픽셀 비율: 25%         어두운 픽셀 비율: 100%
  → 정상                      → 블랙!
```

### 설정 항목 (감지설정 탭)

| 설정 | 기본값 | 쉬운 설명 |
|------|-------|---------|
| 밝기 임계값 | 5 | "얼마나 어두운 점을 까만 점으로 볼 것인가" |
| 어두운 픽셀 비율 임계값 | 98% | "까만 점이 전체의 몇 %이면 블랙인가" |
| 모션 억제 비율 | 0.2% | "어두운 화면이라도 움직임이 조금이라도 있으면 블랙이 아니라고 판단" |
| 몇 초 이상시 알림 발생 | 20초 | "블랙이 이 시간 이상 지속되면 알림" |
| 알림 지속시간 | 60초 | "알림음을 울리는 시간" |

## 3. 스틸 감지 — "화면이 멈췄는가?"

### 기본 원리

200ms(0.2초)마다 화면을 캡처하여, **직전 화면과 현재 화면을 비교**합니다.

```
[감지 순서]

1단계. 직전 화면과 현재 화면의 차이 계산
       ┌──────────────────────────────────────────┐
       │  각 픽셀의 밝기 차이를 구한다              │
       │  차이가 기준값(4) 이상이면 = "변화한 픽셀"   │
       │  차이가 기준값 미만이면 = "변화 없는 픽셀"    │
       └──────────────────────────────────────────┘
       ※ 기준값이 필요한 이유:
         영상 압축(인코딩) 과정에서 미세한 노이즈가 발생하므로
         아주 작은 차이는 실제 움직임이 아닌 노이즈로 걸러냄

2단계. 화면을 3×3 = 9개 블록으로 나눠서 분석
       ┌──────────────────────────────────────────┐
       │  각 블록별로 "변화한 픽셀 비율"을 계산       │
       │  → 9개 블록 중 하나라도 15% 이상 변화하면    │
       │    "화면이 움직이고 있다" → 스틸 아님        │
       │  → 모든 블록이 15% 미만이면                 │
       │    "화면이 멈춰있다" → 스틸                  │
       └──────────────────────────────────────────┘

3단계. 일정 시간 이상 지속되면 알림
       ┌──────────────────────────────────────────┐
       │  스틸이 60초 이상 계속되면 알림 발생         │
       └──────────────────────────────────────────┘
```

### 왜 "블록 방식"인가?

뉴스 선거 개표 화면처럼 **대부분은 정적이지만 한쪽에 수화 통역이 있는 경우**를 생각해보세요.

```
  전체 평균 방식 (이전)                블록 방식 (현재)
 ┌──────────────────────┐           ┌───────┬───────┬───────┐
 │                      │           │       │       │       │
 │   선거 그래프 (정적)   │           │  0%   │  0%   │  0%   │
 │                      │           │ 변화   │ 변화   │ 변화   │
 │                      │           ├───────┼───────┼───────┤
 │                      │           │       │       │       │
 │                      │           │  0%   │  0%   │  1%   │
 │                ┌────┐│           │ 변화   │ 변화   │ 변화   │
 │                │수화││           ├───────┼───────┼───────┤
 │                │통역││           │       │       │ ★★★  │
 │                └────┘│           │  0%   │  0%   │ 35%  │
 └──────────────────────┘           │ 변화   │ 변화   │ 변화   │
                                    └───────┴───────┴───────┘

  전체 평균: 변화 3% → 스틸!          우하단 블록: 35% 변화!
  (수화 영역이 너무 작아서              → 블록 1개라도 15% 초과
   전체 평균에 묻혀버림)                → 스틸 아님 ✓ (정상 판정)
```

### 설정 항목 (감지설정 탭)

| 설정 | 기본값 | 쉬운 설명 |
|------|-------|---------|
| 픽셀당 변화 기준값 | 4 | "얼마나 달라져야 진짜 변화로 인정할 것인가" (인코딩 노이즈 필터) |
| 블록 움직임 임계값 | 15% | "블록 안에서 몇 %가 변해야 움직임으로 인정할 것인가" |
| 타이머 리셋 프레임 수 | 3 | "연속 3번(0.6초) 정상이어야 스틸 타이머를 리셋" (순간 글리치 방지) |
| 몇 초 이상시 알림 발생 | 60초 | "스틸이 이 시간 이상 지속되면 알림" |
| 알림 지속시간 | 60초 | "알림음을 울리는 시간" |

## 4. 블랙과 스틸의 관계

블랙 감지와 스틸 감지는 **독립적으로 작동**하되, 하나의 연결 고리가 있습니다.

```
 ┌─────────────┐     ┌─────────────┐
 │  블랙 감지    │     │  스틸 감지    │
 │             │     │             │
 │ 1. 밝기 판정  │     │ 1. 변화 계산  │──── 변화 비율을 ────┐
 │ 2. 비율 판정  │     │ 2. 블록 분석  │                    │
 │ 3. 모션 억제 ◀├─────┤             │                    │
 │ 4. 시간 판정  │  활용 │ 3. 시간 판정  │                    │
 └─────────────┘     └─────────────┘                    │
                                                        │
                       블랙이 어두운 장면인데               │
                       자막이 스크롤되고 있으면?  ◀──────────┘
                       → 움직임이 있으므로 블랙 취소!
```

즉, 스틸 감지에서 계산한 "변화 비율"을 블랙 감지의 모션 억제에서 참조합니다.
어두운 장면이라도 자막이 스크롤되면 블랙이 아닌 것으로 판단합니다.

## 6. 자주 묻는 질문

### Q. 블랙 감지가 왜 98%인가? 100%면 안 되나?
방송 화면 가장자리에 미세한 밝은 라인이 있을 수 있고, 자막이 순간적으로 나타났다 사라질 수 있습니다. 100%로 하면 이런 경우를 놓칩니다.

### Q. 스틸 감지 60초가 너무 긴 것 아닌가?
뉴스 앵커가 가만히 서 있는 장면, 정적인 자료 화면 등은 수십 초간 변화가 적을 수 있습니다. 너무 짧으면 이런 정상 화면에서도 알림이 울립니다.

### Q. 수화 통역이 있는데도 스틸로 감지되는 경우는?
수화 통역 영역이 매우 작거나 통역사의 움직임이 적을 때 발생할 수 있습니다. 이 경우 "블록 움직임 임계값"을 낮추면(예: 15% → 10%) 더 민감하게 움직임을 잡습니다.

### Q. 설정을 바꾸면 바로 적용되나?
네, 설정 값을 입력하면 즉시 적용됩니다. 프로그램 재시작은 필요 없습니다.
