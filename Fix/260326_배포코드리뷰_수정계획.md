# 배포용 전체 코드리뷰 수정계획

## 문제 현상
- 여러 컴퓨터에 배포를 앞두고 전체 코드 안정성/완성도/최적화 코드리뷰 수행
- core/ 7파일, ui/ 8파일, utils/ 2파일, config/ 2파일 전수 분석
- 재현 조건: 장기 운용, 다른 PC 환경, 네트워크 지연, 장치 분리/재연결 등 다양한 시나리오

## 원인 분석 — 심각도별 분류

### Critical (크래시/데이터 손실 가능)

#### C1. `video_capture.py` — cap.release() 예외 미처리
- 파일: `kbs_monitor/core/video_capture.py` L98-101
- 캡처 카드 비정상 분리 시 `cap.release()` 에서 세그폴트 가능
- cap 객체가 좀비 상태로 유지 → 다음 루프에서 crash

#### C2. `alarm.py` — 사운드 스레드 경합 (race condition)
- 파일: `kbs_monitor/core/alarm.py` L241-242
- `_sound_thread.is_alive()` 체크와 새 스레드 생성 사이 경합
- 알림 빠르게 연속 트리거 시 중복 스레드 생성 → 오디오 혼란

#### C3. `audio_monitor.py` — 스트림 재연결 후 좀비 스트림
- 파일: `kbs_monitor/core/audio_monitor.py` L154-205
- `stream.close()` 후 `stream = None` 미할당
- 재연결 실패 시 이전 좀비 스트림으로 `stream.read()` 재시도 → crash

#### C4. `auto_recorder.py` — 파일명 race condition
- 파일: `kbs_monitor/core/auto_recorder.py` L224-242
- 동일 밀리초에 2개 알림 발생 시 같은 파일명 경합
- 녹화 파일 덮어쓰기 또는 오류

#### C5. `telegram_notifier.py` — 워커 스레드 재시작 경합
- 파일: `kbs_monitor/core/telegram_notifier.py` L133-138
- 여러 스레드가 동시에 `notify()` 호출 시 워커 스레드 중복 생성
- 메시지 중복 전송 또는 손실

#### C6. `telegram_notifier.py` — 큐 가득참 시 알림 무시 (로그 없음)
- 파일: `kbs_monitor/core/telegram_notifier.py` L179-182
- `queue.Full` → `pass` (무시, 로그 없음)
- 운영자가 알림 누락 인지 불가

#### C7. `config_manager.py` / `logger.py` — 상대경로 의존성
- 파일: `kbs_monitor/utils/config_manager.py` L131-133, `logger.py` L18
- `CONFIG_DIR = "config"`, `LOG_DIR = "logs"` 등 상대경로
- 다른 PC/경로에서 실행 시 설정 파일 못 찾거나 잘못된 위치에 생성

### Major (기능 오동작/성능 저하)

#### M1. `main_window.py` — closeEvent 스레드 종료 대기 미흡
- 파일: `kbs_monitor/ui/main_window.py` L1204-1233
- QThread `.stop()` 후 `.wait()` 미호출 → 좀비 스레드 잔류
- 프로그램 종료 시 리소스 미해제

#### M2. `detector.py` — `_prev_frames` / `_near_miss_start` 메모리 누수
- 파일: `kbs_monitor/core/detector.py` L319, L356-365
- ROI 삭제/스틸 감지 비활성화 시 딕셔너리 항목 미정리
- 장기 운용 시 메모리 점진적 증가

#### M3. `signoff_manager.py` — 상태 전환 타이머 리셋 미흡
- 파일: `kbs_monitor/core/signoff_manager.py` L216-228
- 스케줄 변경 시 `_exit_released` 만 리셋, 진입 타이머 미리셋
- 이전 정파 상태가 새 스케줄에 영향

#### M4. `alarm.py` — 볼륨 0 설정 시에도 50% 재생
- 파일: `kbs_monitor/core/alarm.py` L194
- `vol = max(self._volume, 0.5)` → 사용자 의도 무시

#### M5. `auto_recorder.py` — 녹화 큐 오버플로우 시 즉시 중단
- 파일: `kbs_monitor/core/auto_recorder.py` L167
- `_MAX_RECORD_FRAMES` 도달 시 녹화 즉시 중단 → 마지막 부분 손실

#### M6. `main_window.py` — 설정 로드 시 예외 미처리
- 파일: `kbs_monitor/ui/main_window.py` L919-963
- JSON 파싱 오류 시 unhandled exception → 크래시

#### M7. `telegram_notifier.py` — Rate Limit 복구 루프 로직 오류
- 파일: `kbs_monitor/core/telegram_notifier.py` L295-307
- 429 응답 후 재시도 카운트 미증가 → 잠재적 무한 루프

#### M8. `auto_recorder.py` — 런타임 중 고아 임시파일 미정리
- 파일: `kbs_monitor/core/auto_recorder.py` L330-350
- 프로그램 시작 시 한 번만 정리, 런타임 중 생성된 고아파일 누적

### Minor (사용성/유지보수)

#### m1. `video_capture.py` — ret=True, frame=None 유령 프레임 미처리
#### m2. `detector.py` — crop 후 빈 배열 검증 미흡
#### m3. `top_bar.py` — 고DPI 대응 부족 (고정 68px 높이)
#### m4. `roi_editor.py` — HANDLE_RADIUS 고DPI 미대응
#### m5. `settings_dialog.py` — Signal disconnect 누락
#### m6. `requirements.txt` — 버전 상한선 미지정
