## 권장 조치

### 추가 수정 필요 (잔여 Medium)

1. **`AlarmSystem._toggle_blink()` try-except 추가** (`core/alarm.py:232`)
   ```python
   def _toggle_blink(self):
       try:
           self._blink_state = not self._blink_state
           self.visual_blink.emit(self._blink_state)
       except Exception as e:
           import logging
           logging.getLogger(__name__).error("_toggle_blink 오류: %s", e)
   ```
   - `visual_blink` 시그널이 `VideoWidget.set_blink_state()`에 연결되어 있고, 이 슬롯에서 예외가 발생하면 Qt가 이를 `_toggle_blink` 호출 스택으로 전파하여 `_blink_timer` 콜백이 다음 주기부터 중단될 수 있음

### 검증 방법

1. **진입 타이머 stale 수정 검증**: 정파 사이클 1회 완료(IDLE→PREP→SIGNOFF→IDLE) 후 익일 PREPARATION 재진입 시 SIGNOFF 즉시 전환 없이 정상적으로 still_trigger_sec 카운트다운이 시작되는지 확인
2. **_update_summary 예외 보호 검증**: 디버그 로그에서 `_update_summary 오류` 메시지가 발생하지 않는지 5분 로그 확인
3. **장기 운영 검증**: 5분마다 출력되는 `감지 정상 실행 중 (N분 경과)` 로그가 2일 이상 연속 출력되는지 확인
