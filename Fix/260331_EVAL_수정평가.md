## 발견 사항

### Medium

#### M1 — `_on_capture_connected`가 `_config`를 읽어 포트를 저장하므로 근본 문제가 일부 잔존

**위치**: `main_window.py` L255 `_on_capture_connected()`

**내용**:
```python
def _on_capture_connected(self):
    self._active_capture_port = self._config.get("port", 0)
```

`connected` 시그널은 `VideoCaptureThread.run()` L98에서 `cap.isOpened()` 직후 발행된다.
이 시점에서 `video_capture.py`는 내부 `current_port`(뮤텍스로 보호된 `_port` 스냅샷)를 사용해 연결했으나,
`_on_capture_connected()`는 `self._config.get("port", 0)`을 읽는다.

`connected.emit()` → Qt 큐를 통해 메인 스레드로 전달되는 동안 `_on_port_changed()`가 먼저 실행되어
`self._config["port"]`가 이미 새 포트로 변경된 경우, `_active_capture_port`에는 **새 포트 번호가 저장**된다.
그 후 구 포트에서 `disconnected.emit()`이 오면 새 포트 번호로 오류 로그가 찍히는 문제가 재발할 수 있다.

**계획서 원문 인식**: 계획서(L21)에서 이 타이밍 문제를 정확히 지적했으나,
슬롯 구현 시 `self._config.get("port",0)` 의존을 그대로 유지하여 근본 원인이 일부 잔존.

**올바른 수정 방향**: `video_capture.py`의 `connected` 시그널에 현재 연결된 포트 번호를 인자로 포함하거나 (`connected = Signal(int)`), 또는 `_on_capture_connected()`에서 `self._capture_thread._port`를 직접 읽는 방법.

**영향**: 포트 변경 타이밍이 정확히 맞아야 재현되므로 일상 운용에서 발생 빈도는 낮음.

---

#### M2 — `connected` 슬롯에서 성공 로그 미출력

**위치**: `main_window.py` L253~255 `_on_capture_connected()`

**내용**: 기존 `connected` lambda는 "포트 N 연결 성공" 로그를 출력했으나, 교체된 슬롯에서 로그 출력이 제거되었다.

```python
# 기존 (삭제됨)
lambda: self._logger.info(f"SYSTEM - 포트 {self._config.get('port',0)} 연결 성공")

# 교체 후
def _on_capture_connected(self):
    """캡처 스레드 연결 성공 — 연결 시점의 포트 번호를 고정 기록"""
    self._active_capture_port = self._config.get("port", 0)
    # ← 로그 없음
```

현재는 `status_changed` 시그널이 연결되어 `video_capture.py`의 "포트 N 연결 성공" 메시지가 `SYSTEM - 포트 N 연결 성공`으로 출력된다. 따라서 **UI 상의 연결 성공 로그는 유지됨** — 이중 로그 방지 효과도 있다.

**영향**: UI 로그 동작에 실질적 변화 없음. 다만 코드 독자가 슬롯 역할을 오해할 수 있다. docstring이 "로그 미출력" 의도를 명시하지 않음.

### `status_changed` 이중 로그 가능성 검토

`video_capture.py`에서 `status_changed.emit("포트 N 연결 성공")` 발행 → `SYSTEM - 포트 N 연결 성공` UI 출력.
`connected.emit()` → `_on_capture_connected()` → 로그 없음.
**이중 로그 없음 — 정상.**

`disconnected.emit()` → `_on_capture_disconnected()` → `SYSTEM - 포트 N 연결 실패` UI 출력.
`status_changed.emit("포트 N 신호 없음")` 또는 `"포트 N 연결 실패"` → `SYSTEM - 포트 N 신호 없음` UI 출력.

`disconnected` 발생 시 `status_changed`도 함께 발행되므로 **2개의 오류 관련 로그가 출력됨**.

예시 (30프레임 실패 경로):
```
SYSTEM - 포트 0 연결 실패       ← _on_capture_disconnected (error 레벨)
SYSTEM - 포트 0 신호 없음       ← status_changed lambda (info 레벨)
```

이중 출력이지만 메시지 내용이 달라 운영자가 구별할 수 있다. 기능상 문제 없음.
