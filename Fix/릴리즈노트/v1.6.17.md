## 변경사항 요약

### 버그 수정

- **DIAG-AUDIO 오류 고착** (`main_window.py`): `_embedded_alert_start` TOCTOU 방어 (로컬 변수 캡처) + 6개 DIAG 섹션 정상 완료 시 `_diag_last_errors.pop()` 추가 — 장기 운영 후 DIAG-AUDIO 출력 소실 현상 수정
- **alarm 경쟁조건** (`alarm.py`): `_play_sound()`에서 `_stop_sound.clear()` → `threading.Event()` 새 인스턴스 교체 — 설계 원칙 위반 수정
- **alarm 스레드 시작 미보호** (`alarm.py`): `play_test_sound()` `start()` OSError try-except 추가
- **telegram 워커 IDLE 방치** (`telegram_notifier.py`): `ensure_worker_alive()` 추가 + DIAG-TELEGRAM 30초 주기 호출 — 알림 없는 구간에서 워커 사망 시 미복구 수정
- **recorder ffmpeg 병렬 실행** (`auto_recorder.py`): `trigger()` 이전 스레드 alive 체크를 상태 변경 이전으로 이동 — ffmpeg 합성 중 새 녹화 시작 시 상태 오염 방지

### 개선

- telegram `notify()` 내 워커 재시작 인라인 코드 → `ensure_worker_alive()` 단일화 (중복 제거)
- `ensure_worker_alive()` 로그 메시지에서 "(DIAG 주기)" 제거 (호출 경로 무관한 표현)
- `auto_recorder.py` `trigger()` `is_alive()` 체크 의도 주석 추가
