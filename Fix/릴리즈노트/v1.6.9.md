# v1.6.9 릴리즈 노트

**릴리즈 날짜**: 2026-03-31

## 변경사항 요약

### 버그 수정
- 포트 연결 실패 로그에 실제 포트 번호가 정확히 표시되지 않던 문제 수정
  - `VideoCaptureThread.status_changed` 시그널이 `main_window.py`에서 연결되지 않아 연결 성공/실패 메시지가 UI에 표시되지 않던 문제 해결
  - `connected` / `disconnected` lambda를 전용 슬롯(`_on_capture_connected`, `_on_capture_disconnected`)으로 교체 — 포트 변경 타이밍 이슈로 다른 포트 번호가 찍히던 문제 방지
- 텔레그램 전송 연속 실패 시 오류 로그 스팸 억제
  - 방화벽 환경에서 알림 발생마다 2~3개의 오류 로그가 UI를 채우던 문제 개선
  - 1~3회 실패: 기존대로 UI에 표시
  - 4회 이상: 파일 로그 전용 (UI 미표시)
  - 10·20·30… 배수: 요약 메시지 1회 UI 표시 (`"텔레그램 N회 연속 실패 중 — 네트워크/방화벽 확인"`)
  - 비-429 HTTP 오류 및 재시도 중간 로그에도 동일한 억제 로직 적용
  - 오류 유형 분류: `ConnectionError` → "네트워크 차단", `Timeout` → "응답 시간 초과"
